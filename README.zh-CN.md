# 书签流整理工坊

[English](README.md) | 简体中文

面向 Chrome 书签重度用户的整理扩展：导出、导入、A/B 对比、时间修复、模板化 AI 提示词、树形批量编辑，集中在一个工作流里完成。

## 适用场景

- 迁移浏览器后，想批量清洗与重组书签。
- 需要把书签结构交给 AI，按模板生成可复用提示词。
- 想以 A/B 集对比定位差异，并按时间策略修复标准 HTML 书签。
- 想在不离开扩展的情况下直接做树形编辑与批量操作。

## 核心能力

### 1. 导出与提示词

- 读取 Chrome 全量书签树。
- 按文件夹选择导出，支持完整/精简 JSON。
- 生成包含 YAML 书签数据的 AI 提示词。
- 支持模板管理、编辑、切换生效模板。
- 支持哈希模式提示词与哈希导入解析。

### 2. 导入与保护

- 支持 JSON/YAML 导入并重建书签结构。
- 导入前可自动备份，降低误操作风险。
- 导入默认新建根目录，避免覆盖原始书签树。

### 3. A/B 对比与检索

- 支持 A/B 选择集对比。
- 对比结果支持排序与筛选。
- 提供独立对比搜索页，便于快速定位条目。
- 在关键书签条目视图中展示添加时间，便于判断来源与新旧。

### 4. 标准 HTML 时间修复

- 时间修复功能已独立为单页，不占用默认对比页。
- 以 B 为目标集，仅修复匹配 URL 的时间字段，不删改 B 的其他内容。
- 保持导出顺序与 B 一致。
- 同 URL 多条目场景下，可按 B 条目粒度选择 A 的时间来源。

### 5. 树形编辑

- 搜索与定位书签节点。
- 安全打开链接。
- 新建文件夹/书签。
- 多选、拖拽、复制/剪切/粘贴与批量操作。

### 6. 体验与国际化

- 中英文界面（zh-CN / en-US）。
- 主题支持浅色/深色/跟随系统。

## 技术栈

- Plasmo (Manifest V3)
- React 18 + TypeScript
- react-arborist
- yaml

## 快速开始

1. 安装依赖

```bash
npm install
```

2. 开发模式运行

```bash
npm run dev
```

3. 在 Chrome 打开 `chrome://extensions/`，启用开发者模式，加载生成目录（通常为 `build/chrome-mv3-dev`）。

## 构建与打包

```bash
npm run build
npm run package
```

## 权限说明

- `bookmarks`：读取与写入书签树。
- `downloads`：导出/备份文件下载。
- `storage`：保存模板与页面状态。

## 导入格式示例

数组结构：

```json
[
  {
    "title": "技术",
    "children": [{ "title": "MDN", "url": "https://developer.mozilla.org" }]
  }
]
```

对象结构（含 `children`）：

```json
{
  "title": "Root",
  "children": [{ "title": "Google", "url": "https://www.google.com" }]
}
```
